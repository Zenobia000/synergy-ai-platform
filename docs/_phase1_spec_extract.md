## 一句話定位

教練版成交副駕駛：用 4 週跑通最短成交閉環，不做完整平台。

## 核心流程

Lead 入口 → 三階段 AI 問卷 → AI 摘要（客戶版 + 教練版）直接 email 分享 → 產品/SKU 初步建議（for 教練）→ 商談話術（前/中/後 for 教練）→ 48h/7d/30d 跟進提醒（連動 Google 日曆）

## 主要使用者

| 角色 | 在 MVP 中做什麼 |
| :--- | :--- |
| 教練 / 經銷商 | 每天邀約客戶填問卷、商談前讀 AI 摘要、商談後依提醒跟進 |
| 客戶 / 潛在客戶 | 透過連結填問卷、取得非醫療式健康關注摘要 |
| Leader / Pilot 管理者 | 查看 3-5 位教練的使用數據、成交流程與問題（Leader Summary 頁） |

## In-scope 模組（本 MVP 必做）

| 模組 | 範圍 |
| :--- | :--- |
| **M1 基礎 Lead / 問卷入口** | 問卷連結、邀請文案、私訊開場草稿、來源欄位、合規初篩 |
| **M2 三階段 AI 問卷** | Phase 1 快速分流、Phase 2 六大核心、Phase 3 動態追問、雙版本摘要 |
| **M3 商談副駕駛** | 商談前 5 分鐘摘要、開場、產品銜接、異議、柔性成交、商談後訊息 |
| **M4 基礎 CRM 與跟進提醒** | 自動建檔、十種狀態、48h/7d/30d 提醒、LINE/WhatsApp/Email 草稿 |
| **合規 AI（強制路徑）** | 醫療/收入/誇大/金字塔風險檢查、改寫、HITL |
| **M6 輕量 Activity Tracking** | 教練數據、跟進執行率、AI 摘要使用、Leader Summary、新手教練進度 |

## Out-of-scope（本 MVP 明確不做）

- 完整社群內容引擎、自動發文、SEO
- 完整 CRM 自動化、回購預測、流失預測
- 完整 M5 見證與轉介紹系統
- 完整 M6 團隊管理、組織樹、獎金或業績串接
- 完整電商、佣金計算、公司級 BI
- 深度官方 API 串接（Exigo / Synergy 後台等）

## M1 — 基礎 Lead / 問卷入口

| # | 功能 | 描述 |
| :--- | :--- | :--- |
| F1.1 | 產生問卷連結 | 每位教練可一鍵產生帶教練識別碼的問卷連結 |
| F1.2 | 問卷邀請文案產生 | AI 依客戶來源 / tag 產生邀請文案 |
| F1.3 | 私訊開場草稿 | LINE / WhatsApp 開場白草稿 |
| F1.4 | 客戶來源欄位 | LINE、社群、朋友介紹、活動、其他 |
| F1.5 | 基本合規檢查 | 邀約文案在送出前自動跑合規檢查 |

## M2 — 三階段 AI 問卷

### 問卷結構

| 階段 | 名稱 | 內容 |
| :--- | :--- | :--- |
| Phase 1 | 快速分流 | 年齡、生活型態、主要健康關注 |
| Phase 2 | 六大核心 | 壓力、睡眠、消化、手腳冰冷、水腫、排便 |
| Phase 3 | 動態追問 | 依回答顯示最相關追問（飲食、水分、活動、體態、女性週期、補充品/藥物） |

### AI 處理輸出

| # | 功能 | 描述 |
| :--- | :--- | :--- |
| F2.1 | 自動計分 | 健康關注標籤 + 權重 + 前三大優先關注 |
| F2.2 | 客戶版摘要 | 健康關注與生活習慣方向，**不使用診斷語言** |
| F2.3 | 教練版摘要 | 商談切入點、紅旗提醒、產品/SKU 初步建議 |
| F2.4 | 30 秒內輸出 | 完成提交後 30 秒內必須產出兩版摘要 |

## M3 — 商談副駕駛

| 階段 | 功能 |
| :--- | :--- |
| 商談前 | 5 分鐘摘要、核心痛點三句話、建議開場與暖場問題 |
| 商談中 | 產品銜接話術、可能異議與建議回覆、柔性成交邀約 |
| 商談後 | 下一步訊息草稿（自動帶入跟進排程） |

## M4 — 基礎 CRM 與跟進提醒

### 客戶狀態（10 種）

```
新名單 → 已填問卷 → 已預約 → 已商談 → 已推薦
       → 試用中 → 已成交 / 未成交 → 需回訪 → 沉默
```

### 提醒節點

| 節點 | 觸發條件 | 預設動作 |
| :--- | :--- | :--- |
| 48 小時 | 商談 / 推薦後未成交 | 產出 LINE/WhatsApp/Email 跟進草稿 |
| 第 7 天 | 試用中或冷淡 | 試用回饋詢問草稿 |
| 第 30 天 | 沉默 / 未回購 | 喚醒草稿 |

### 功能清單

| # | 功能 | 描述 |
| :--- | :--- | :--- |
| F4.1 | 客戶自動建檔 | 問卷完成自動建立 Customer |
| F4.2 | 狀態管理 | 10 種狀態流轉（手動 / 自動） |
| F4.3 | 最後聯繫時間與備註 | 每個 customer 必填欄位 |
| F4.4 | 跟進草稿生成 | LINE / WhatsApp / Email 三通道 |

> **連動**：48h/7d/30d 提醒需連動 Google 日曆。

## 合規 AI（強制路徑）

所有 AI 對外訊息必須先通過合規檢查。

### 處理流程

1. 文字進入 Compliance Service
2. 規則庫初篩 → LLM 二次覆核
3. 高風險 → 標記 + 安全改寫 + HITL 人工確認
4. 低風險 → 自動加上免責聲明

### 檢查類型

| # | 類型 | 範例 |
| :--- | :--- | :--- |
| C1 | 醫療宣稱 | 「治療」、「治癒」、「預防 X 病」 |
| C2 | 收入宣稱 | 「月入 X 萬」、「保證收入」 |
| C3 | 誇大效果 | 「100% 有效」、「立即見效」 |
| C4 | 金字塔風險語句 | 「拉人頭」、「上線抽成」 |

- Leader 進入 Leader Summary 頁

## 資料模型（Phase I 必要 Entity）

| Entity | 主要欄位 |
| :--- | :--- |
| User | id, 姓名, 角色（coach / leader / admin）, 品牌, 語言, 上線 / leader id |
| Customer | id, 姓名 / 暱稱, 聯絡方式, 來源, 狀態（10 種）, 教練 id, 同意紀錄 |
| QuestionnaireResponse | 答案 JSON, 分數, 健康標籤, 優先關注, 紅旗提醒 |
| Recommendation | SKU, 推薦理由, 注意事項, 人工調整紀錄 |
| ConversationPlan | 商談摘要, 話術（前/中/後）, 異議處理, 跟進訊息 |
| FollowUpTask | 時間, 狀態, 提醒內容, 是否完成 |
| ComplianceLog | 原文, 風險類型, 安全改寫, 審核狀態 |
| EventLog | 使用者行為, 時間, 功能, 結果（含 latency/token/模型版本） |

### 關聯

```
User (coach) ──< Customer ──< QuestionnaireResponse
                    │              │
                    │              ▼
                    │         Recommendation
                    │
                    ├──< ConversationPlan
                    └──< FollowUpTask

ComplianceLog ──→ 任何對外訊息（一比一保留：原文 / 改寫後 / 風險等級 / 是否人工覆核）
EventLog      ──→ 任何使用者行為
```

## AI 架構

### 服務拓撲

```
Web App / LINE Bot 入口
        │
Workflow（M1 → M2 → M3 → M4 → M6）
        │
LLM Orchestrator（任務路由）
   │           │              │
   ▼           ▼              ▼
 RAG       Compliance      Scoring
（產品/    Service        （問卷+標籤）
 話術庫）
        │
        ▼
   HITL（SKU / 高風險話術）
        │
        ▼
   Event Log + 個人記憶
```

### 模型分層（Phase I 成本控管）

| 任務 | 模型等級 | 範例 |
| :--- | :--- | :--- |
| 摘要、分類、私訊草稿 | 低成本模型 | 客戶摘要卡、跟進草稿、邀約文案 |
| 商談話術、動態追問 | 中階模型 | M2 Phase 3、M3 商談中提示 |
| 合規最終覆核、紅旗判斷 | 高階模型 | Compliance 二次覆核、SKU 推薦 |

### 強制路徑

- 所有對外訊息 → 必過 Compliance Service
- SKU 推薦、見證、健康建議 → 必過 HITL
- 任何 AI 任務 → 必寫 EventLog

## 非功能需求

| 類別 | 需求 |
| :--- | :--- |
| 效能 | 問卷送出後摘要 ≤ 30 秒；商談前摘要 ≤ 5 秒；跟進草稿 ≤ 3 秒 |
| 可用性 | Pilot 期 ≥ 99.0% uptime |
| 可觀測性 | 所有 AI 任務寫 EventLog（latency、token、模型、風險詞） |
| 多語系 | Phase I：繁中（介面與輸出皆繁中） |
| 權限 | 教練只能看自己客戶；Leader 看下線匯總，不見 PII |
| 資料保護 | Customer PII 加密儲存；同意紀錄必填 |

## 驗收標準

| # | 標準 |
| :--- | :--- |
| AC1 | 客戶可完成問卷，系統可自動建立客戶資料 |
| AC2 | 問卷完成後 30 秒內產出客戶版摘要與教練版摘要 |
| AC3 | 所有 AI 對外訊息必須通過合規檢查 |
| AC4 | 教練可看到客戶狀態、問卷結果、商談建議與跟進任務 |

## 北極星指標

| 指標 | 目標 |
| :--- | :--- |
| 問卷完成率 | ≥ 50% |
| 問卷 → 預約商談轉換率 | ≥ 30% |
| 商談 → 成交率 | ≥ 20% |
| 教練商談準備時間 | ≤ 5 分鐘 |
| 48 小時未成交跟進執行率 | ≥ 80% |

## 4 週開發排程

```
W1 ─ 骨架
   • 問卷流程（Phase 1/2/3）
   • 計分規則 + Customer schema
   • 產品知識庫初版（RAG）
   • Compliance 規則庫 v1

W2 ─ AI 核心
   • M2 雙版摘要（客戶版 + 教練版）
   • M3 商談前 / 中 / 後話術
   • SKU 初步推薦 + HITL 流程
   • Compliance Service（強制路徑）

W3 ─ 跟進閉環
   • M4 基礎 CRM + 10 種狀態
   • 48h / 7d / 30d 提醒排程
   • LINE / WhatsApp / Email 草稿
   • M6 activity tracking + Leader Summary
   • 內部測試

W4 ─ Pilot
   • 3-5 位教練 pilot 上線
   • 每日數據檢查與快速迭代
   • 高風險話術修正
   • 是否進入 Phase II 決策評審
```

## 風險

| # | 風險 | 緩解 |
| :--- | :--- | :--- |
| R1 | 合規踩線 | 規則庫 + LLM 覆核 + HITL，三道防線 |
| R2 | Pilot 教練不用 | 只做每天會用的成交閉環；W2 起 weekly 1:1 |
| R3 | 資料量不足做訓練 / 評估 | EventLog 從 D1 啟用，4 週累積基準資料 |
| R4 | 時間壓力下品質下降 | M5 / M6 完整版砍掉，只做核心；HITL 流程要簡單但要存在 |
| R5 | AI 成本超預期 | 摘要 / 分類用低成本模型；高階模型只用在合規與商談 |
| R6 | Brand 寫死風險 | 即便 Phase I，產品庫 / 話術庫 / 合規詞庫都用資料驅動 |
