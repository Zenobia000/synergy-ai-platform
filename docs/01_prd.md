## 0. 修訂說明（v2.0 → v3.0）

本 PRD 依據 2026-05-06 客戶 Phase I MVP 規格書升級為 v3.0，主要變動為：

- **模組擴充**：從 4 功能閉環（v2.0）擴為 6 大模組完整閉環（M1-M4 + 合規 AI + M6）
- **Epic 重組**：從 4 個 Epic (A-D) 擴為 6 個 (A-F)，新增商談副駕駛前/中/後、10 種客戶狀態、合規強制路徑、Leader Summary

- **合規 AI 晉升**：從 Won't 晉升為 Must（規則庫 + LLM 二次覆核 + HITL）
- **HITL 新增**：SKU 推薦、見證、健康建議需人工審核確認
- **Leader 視角新增**：Activity Tracking + Leader Summary + 新手教練進度
- **Google Calendar 整合**：提醒自動建立日曆事件
- **前端框架改為 React 19 + Vite**（ADR-013，推翻 ADR-001 前端決策）

詳見 [12_phase1_mvp.md](./12_phase1_mvp.md) 之「v2.0 → v3.0 差異對照」；ADR 見 [03_adr.md](./03_adr.md) ADR-013。

## 2. 商業目標與 OKRs

### 背景與痛點

健康直銷事業依賴人際信任成交，但中段教練（非頂尖 20%）的成交率普遍低於 15%，原因是：
1. **商談前缺乏結構化客戶資訊**（客戶健康優先順序、可推薦產品、預期異議無人提示）
2. **商談後忘記跟進**（無自動提醒，只靠教練記憶，48h 內流失率 > 60%）
3. **新教練缺乏話術**（無系統化商談前/中/後話術，只能自己摸索或問資深教練）
4. **無法看下線進度**（Leader 層級無工具觀察 Pilot 教練的真實使用狀況）

現有流程仰賴手工 App Script 問卷 + Google Sheet + 教練記憶，80% 資訊在商談後 48 小時內流失。

### 策略契合度

- **短期**：驗證「AI 副駕駛能否把中段教練成交率從 15% 拉到 20-25%；商談準備從 30+ 分鐘縮短到 5 分鐘」，為後續投資背書。
- **中期**：把 Phase I 沉澱為跨品牌 SaaS 平台核心（M2 問卷 + M3 商談 + M4 CRM），預留 multi-tenant 架構。
- **長期**（18 個月）：擴張至 USANA、Herbalife、Amway 等其他健康直銷品牌。
- **合規基礎**：建立強制合規路徑（規則庫 + LLM + HITL），為未來跨品牌擴張提供合規護欄。

### OKRs（MVP 上線後 8 週內驗證）

| Objective | Key Result | 衡量方式 | 目標值 |
| :--- | :--- | :--- | :--- |
| **O1. 驗證 AI 副駕駛能提升教練生產力** | KR1: 商談成交率 | Pilot 教練實測對照 | ≥ 20%（基線 15%） |
| | KR2: 商談準備時間 | 教練回報 | ≤ 5 分鐘（基線 30+ 分鐘） |
| | KR3: 48h 跟進執行率 | 系統提醒送達 + 回報 | ≥ 80%（基線 < 40%） |
| **O2. 驗證使用者採用** | KR1: 問卷完成率 | 系統埋點 | ≥ 50% |
| | KR2: 問卷 → 預約商談轉換率 | 系統埋點 | ≥ 30% |
| **O3. 建立可擴張技術底座** | KR1: 資料 schema 已標記 tenant_id | Code review | 100% 核心表 |
| | KR2: LLM 抽象層可切換 ≥ 2 model | 工程驗證 | Gemini + Claude 至少一次雙跑 |

**Phase II 解鎖條件**：5 個 KR 中至少 3 個達標，且至少 1 位 Pilot 教練回報「願意付費續用」；合規檢查誤判率 < 5%。

## 3. 目標用戶（Personas）

### Persona 1: 中段教練「阿明」（主要 MVP 使用者）

| 項目 | 描述 |
| :--- | :--- |
| **角色** | Synergy WorldWide 經營者，入行 1-3 年，月業績 2-5 萬 NTD |
| **痛點** | 1. 商談前不知道該問什麼、產品該推什麼<br>2. 商談後常忘記 48 小時內跟進<br>3. 不知道客人說「我再想想」後該怎麼辦<br>4. 客戶填了健康問卷但自己沒時間研究該推什麼 |
| **目標** | 靠系統化工具把成交率提升、把跟進變成自動化、話術有人提示 |
| **行為** | 每週 3-5 次商談，全程 LINE + 手寫筆記，沒 CRM；隨時用手機；需要「秒懂」的摘要 |

### Persona 2: 潛在客戶「王小姐」（問卷填答者）

| 項目 | 描述 |
| :--- | :--- |
| **角色** | 被朋友（教練）邀請填問卷的潛在客戶，35-55 歲 |
| **痛點** | 1. 不想一次揭露太多健康資訊<br>2. 不知道自己適不適合、會不會被硬推銷<br>3. 看不到填問卷對自己有什麼好處 |
| **目標** | 了解自己健康狀態、拿到可參考的建議、保有自主決定權 |
| **行為** | 手機優先、5-10 分鐘注意力、LINE 是主要溝通管道；需要「友善、非醫療語言」的摘要 |

### Persona 3: 資深 MEGA「老陳」（Leader / 管理者，本 MVP 觀察者 + 部分使用者）

| 項目 | 描述 |
| :--- | :--- |
| **角色** | 10+ 年 MEGA 經營者，月業績 30+ 萬，帶 3-5 位下線教練 |
| **痛點** | 看不到下線的真實進度、無法快速發現「誰卡在商談未成交」需要 1:1 coaching、新手教練進度無人追蹤 |
| **目標** | 把下線進度系統化可視、找出需要幫助的教練、支持新手教練成長 |
| **行為** | 週一早會、會用簡單的資料板；需要「一頁紙摘要」看全體下線狀況 |

## 4. 競爭分析

| 維度 | 手工（App Script + Sheet） | Airtable/Softr 通用 CRM | Flashy / Modere Pro | **Synergy AI（我們）** |
| :--- | :--- | :--- | :--- | :--- |
| **核心功能** | 問卷 + 手動記錄 | 表單 + 資料庫 | MLM 專用 CRM | 問卷 + AI 商談摘要（前/中/後）+ 自動跟進 + Leader Summary |
| **AI 能力** | 無 | 無 | 有限 | Gemini 驅動研判 + 商談副駕駛 + 合規檢查 |
| **定價** | 免費 | 10-30 USD/人/月 | 30-80 USD/人/月 | 待定（Pilot 期免費） |
| **優勢** | 零成本 | 靈活 | 成熟、健康導向 | **商談前/中/後摘要是業界唯一**；内建合規 |
| **劣勢** | 無自動化 | 要自建邏輯 | 不懂直銷成交閉環 | 初期資料少；合規詞庫需磨合 |
| **差異化機會** | — | — | — | 「從問卷到成交的完整閉環」+「教練 5 分鐘即可準備商談」 |

### 風險登記

| ID | 風險 | 影響 | 機率 | 緩解策略 |
| :--- | :--- | :--- | :--- | :--- |
| R-01 | LLM 摘要品質不足以提升成交率 | HIGH | MED | Pilot 第 2 週做 A/B，若無提升則升級到 Claude Sonnet/Opus |

| **R-09** | **HITL 人工審核成為瓶頸，拖累上線時程** | MEDIUM | MED | 自動化低風險案例（≥ 80%）；高風險限時審核（< 30 min）；無共識案例記錄備檔，改寫後重審 |

### 附錄 B：術語表

| 術語 | 定義 |
| :--- | :--- |
| Pilot | 試點，指 MVP 階段選定的 3-5 位教練 |
| Closer's Copilot | 教練成交副駕駛，本專案的產品內部代號 |
| 商談前摘要 | 教練打開商談前看的單頁 AI 摘要（核心功能） |
| 雙版摘要 | 問卷完成後系統產生的兩份摘要：客戶版（友善、可分享）+ 教練版（含紅旗、推薦、切入點） |
| 健康等級 | 0-100 分的問卷研判結果（對應前三大優先關注） |
| 紅旗警訊 | 問卷偵測到的健康風險訊號或需特別關注的項目 |
| HITL | Human-In-The-Loop，人工審核環節（高風險話術、SKU 推薦、見證需人工確認） |
| ComplianceService | 合規檢查服務（規則庫初篩 + LLM 二次覆核 + 改寫 + HITL 觸發） |
| tenant_id | Multi-tenant 資料表欄位，MVP 只有一個 Synergy 值，為跨品牌擴張預留 |
| LiteLLM | Python 套件，抽象化多家 LLM 供應商 API（Gemini、Claude、GPT） |
| EventLog | 系統埋點日誌，記錄每次 AI 任務（延遲、token 計數、模型、風險詞觸發） |
| OnboardingTask | 新手教練進度 checklist 項目（如「使用 briefing 3 次」「完成首個成交」） |
| Leader Summary | Leader 視角資料板，展示下線教練本週漏斗 + 新手進度 |
| MEGA | 客戶內部對資深經營者的稱呼 |
| Vite | 前端建置工具，快速開發與極小化生產輸出 |
| React Router | React 路由庫（v7），用於 SPA 路由管理 |
| Cloudflare Pages | 邊緣計算部署平台，支援靜態網站與無伺服器函式 |
