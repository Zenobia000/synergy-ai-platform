## 1. Domain Know-How（產品與業務知識）

### 1.1 🔴 產品庫（必交付，影響 M2 SKU 推薦 + RAG）

**用途**：AI 依據問卷答案推 SKU，必須有完整產品資料庫（建議 Excel/Google Sheet 或結構化文件提供）。

| 欄位 | 必填 | 範例 |
| :--- | :---: | :--- |
| `sku` | ✅ | `SYN-PROBIO-30` |
| `中文品名` | ✅ | 益生菌複方膠囊 |
| `英文品名` | ✅ | Probiotic Complex |
| `主成分` | ✅ | 5 株益生菌、益生元 FOS |
| `針對問題標籤` | ✅ | 消化、便秘、免疫 |
| `建議用法` | ✅ | 每日 1 顆，餐後溫水服用 |
| `禁忌與注意事項` | ✅ | 免疫抑制患者請諮詢醫師 |
| `不建議組合` | ⚪ | 與抗生素間隔 2 小時 |
| `價格 NTD` | ✅ | 1,200 |
| `規格` | ✅ | 30 顆/瓶 |
| `產品文案 / 賣點` | ✅ | 50 字內賣點摘要 |
| `合規禁用詞` | ⚪ | 不得宣稱「治療便秘」 |

**Pilot 期最少**：20 個 SKU 完整資訊。

### 1.2 🔴 話術庫（必交付，影響 M3 商談副駕駛）

| 類別 | 內容 | 量 |
| :--- | :--- | :---: |
| **開場話術** | 不同來源（LINE 邀約、活動、朋友介紹）的開場 | ≥ 5 種 |
| **產品銜接話術** | 從問卷痛點切到產品推薦的橋接句 | ≥ 10 條 |
| **異議處理庫** | 常見異議 + 標準回覆 | ≥ 15 對 |
| **柔性成交話術** | 不施壓的下單邀約 | ≥ 5 種 |
| **跟進話術範本** | 48h / 7d / 30d 三節點 | 每節點 ≥ 3 種 |

**格式建議**：每條話術標 `(產品 SKU | 適用情境 | 關鍵字)` 三標籤，便於 AI 檢索。

### 1.3 🔴 合規詞庫 C1-C4（必交付，影響 ComplianceService）

對應 [10_security.md §13.1](./10_security.md)。客戶法務 / 合規團隊提供：

| 類別 | 詞彙 | 量 |
| :--- | :--- | :---: |
| **C1 醫療宣稱** | 治療、治癒、預防 + 病名清單（如：糖尿病、高血壓、癌症...）| ≥ 60 條 |
| **C2 收入宣稱** | 月入 X 萬、保證收入、被動收入、財富自由... | ≥ 40 條 |
| **C3 誇大效果** | 100% 有效、立即見效、神奇... | ≥ 40 條 |
| **C4 金字塔風險語句** | 拉人頭、上線抽成、層級獎金、總公司... | ≥ 40 條 |

**最少**：每類 40 條，總計 ≥ 200 條。每條附「替代用語」（用於自動改寫）。

**格式範例 YAML**：
```yaml
C1_medical_claims:
  - phrase: "治療糖尿病"
    severity: high
    suggested_rewrite: "支持血糖管理"
  - phrase: "預防癌症"
    severity: high
    suggested_rewrite: "強化日常保健"
```

### 1.4 🔴 問卷題目最終版（必交付，影響 M2 全部）

對應 [12_phase1_mvp.md §M2](./12_phase1_mvp.md)。三階段都要：

| 階段 | 題數 | 內容 | 計分規則 |
| :--- | :---: | :--- | :--- |
| **Phase 1 快速分流** | 5 題 | 年齡、生活型態、主要健康關注 | 標籤 + 分流路徑 |
| **Phase 2 六大核心** | 12 題 | 壓力、睡眠、消化、手腳冰冷、水腫、排便 | 每題 0-3 分，加總 |
| **Phase 3 動態追問** | 3-8 題 | 飲食、水分、活動、體態、女性週期、補充品/藥物 | 依 Phase 2 分數路由 |

**必交付**：
- [ ] 題目原文（中文）
- [ ] 答案選項
- [ ] 計分權重表（每題對應的健康標籤與分數）
- [ ] 紅旗判定規則（什麼答案組合觸發紅旗，例：「胸痛 + 高血壓家族史」→ 紅旗）
- [ ] Phase 3 動態路由表（Phase 2 答 X 則顯示 Phase 3 第 Y 題）
- [ ] 健康關注 Top 3 排序邏輯

### 1.5 客戶版 vs 教練版摘要範本

| 版本 | 用途 | 範例段落 |
| :--- | :--- | :--- |
| 客戶版 | 寄給客戶看，**禁用診斷語言** | 「您的睡眠品質可改善，建議晚間 10 點前準備休息...」 |
| 教練版 | 教練商談前看，含紅旗、SKU 推薦理由 | 「紅旗：壓力分數 9/12 + 睡眠分數 8/12，建議切入 SYN-RELAX-30，引用 Q5 答案...」 |

客戶提供 **2-3 篇歷史摘要範本**（含真實案例脫敏），讓 AI 學習語氣與結構。

## 2. 帳號與授權

### 2.1 Synergy 內部帳號（如有）

- [ ] 客戶後台只讀帳號（如要對齊產品價格 / 庫存，僅 Phase II 需要）
- [ ] **Phase I 不需要 API 串接**（依規格 4.2 Out-of-scope）

### 2.2 🔴 LINE Official Account（W0 D1 啟動申請）

對應 ADR-008、[01_prd.md R-06](./01_prd.md)。客戶需提供：

- [ ] 商家證明文件（公司登記、營業項目）
- [ ] 客服窗口聯絡資訊
- [ ] OA 名稱與大頭貼（品牌識別）
- [ ] 預設歡迎訊息文案
- [ ] **客戶或客戶指定人作為 OA 擁有者**（Channel Owner）
- [ ] 收到 LINE 審核通過信後，把 Channel Secret + Access Token 交給開發團隊

**審核期約 1-2 週**，務必 W0 D1 啟動。

### 2.3 🔴 Google Workspace（W1 D1 前）

對應 [03_adr.md ADR-008/012](./03_adr.md)、[10_security.md §14](./10_security.md)。

- [ ] Google Cloud Console 專案（客戶或我方擁有）
- [ ] OAuth 2.0 Client ID / Secret（「網頁應用程式」型別）
- [ ] OAuth consent screen 通過（內部測試版即可，Pilot 期 < 100 用戶）
- [ ] Authorized redirect URI（指向我方 API：`https://api.synergy-ai.tw/integrations/google/oauth/callback`）

### 2.4 LLM 供應商（建議我方代開）

- 預設 Gemini API（成本低）
- 客戶若要求用自家 OpenAI/Claude → 提供 API Key
- 預算上限 1,500 NTD/月（Pilot 期）

### 2.5 Email 寄送（Resend）

- [ ] 寄件網域 `noreply@<客戶網域>` 的 DNS 寫入權限（SPF/DKIM/DMARC 設定）
- [ ] 或客戶接受用我方網域 `noreply@synergy-ai.tw`

## 3. 人員（Pilot 與 HITL）

### 3.1 🔴 Pilot 教練（3-5 位）

對應 [01_prd.md Q-001](./01_prd.md)。組成建議：

| 級別 | 人數 | 用途 |
| :--- | :---: | :--- |
| MEGA / 資深 | 1 位 | 觀察 Leader Summary 視角 |
| 中段教練 | 2-3 位 | 主要 user，驗證成交率提升 |
| 新手教練 | 1 位 | 驗證 Onboarding 流程 |

**每位需提供**：
- [ ] 姓名、聯絡方式
- [ ] LINE ID
- [ ] Email
- [ ] 同意書簽妥（明示資料用於 AI 分析、實驗性產品、可隨時退出）
- [ ] 願意連續配合 4 週、每週訪談 30 min

### 3.2 🔴 HITL Reviewer（Q-006，建議 Pilot 期我方代審）

兩種模式擇一：

**A. 客戶內包**（Phase II 推薦）
- [ ] 指派 1-2 位客戶內部合規 / 法務人員
- [ ] 簽 NDA + 系統使用培訓 30 min
- [ ] 工作日 9:00-18:00 待命，30 min SLA
- [ ] 後備：超時 escalation email 收件人

**B. 我方代審**（Pilot 期建議）
- [ ] 客戶授權我方代審
- [ ] 簽署「合規責任範圍書」釐清：誤過導致客訴 → 客戶承擔產業合規後果，我方承擔流程瑕疵責任
- [ ] 客戶提供「邊界案例庫」（10-20 則歷史誤觸案例）讓我方培訓 Reviewer

### 3.3 種子客戶名單（W3 內測用）

- [ ] 10-20 位實際潛在客戶
- [ ] Pilot 教練自行邀請即可（不必客戶總部統一準備）

## 5. 法務 / 隱私

### 5.1 🔴 隱私權政策（W1 D5 前上線）

對應 [11_deployment.md §20](./11_deployment.md)。客戶法務需確認：

- [ ] 客戶方願意以「Synergy WorldWide」名義作為個資控制者（data controller）
- [ ] 我方為「個資處理者」（data processor），簽 DPA（資料處理協議）
- [ ] 隱私政策草本（中文）— 必須明示：
  - 蒐集的資料範圍（健康問卷答案、聯絡方式）
  - 用途（AI 分析、商談支援、合規檢查、HITL 人工覆核）
  - 保留期（ComplianceLog 5 年；其他資料用戶可請求刪除）
  - 跨境傳輸（LLM 可能在境外，需告知）

### 5.2 🔴 同意條款文字

- [ ] 問卷頁「我同意 [條款連結] 並理解此非醫療診斷」勾選框文案
- [ ] Customer 同意紀錄存證欄位（IP、時間戳、版本號）

### 5.3 ComplianceLog 5 年保留同意

- [ ] 客戶法務 sign-off 接受「即便客戶要求刪除資料，ComplianceLog 仍保留（去識別化）」
