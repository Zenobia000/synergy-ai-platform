## 1. 一句話描述

**「直銷商的關懷 Copilot — 記得每位你關心的人、知道今天該聯絡誰、講話有溫度不像在推銷。」**

Phase I 在 4 週內交付 **11 個工具**給 5 位 Synergy 教練 + 1 位 Leader + 5 位下線做設計夥伴 pilot；W4 結尾以「至少 4 位教練自願付 $39」作為 PMF 的真實證據。

> 本 PRD 是 v7（2026-05-16）方向。舊 v6 PRD（品牌總部買單、健康問卷主流程）已 pivot 廢棄；工程深度章節抽出為 Phase II/III 藍圖，**不在 Phase I 範圍內**。

## 3. 核心功能模組（11 個工具）

> 每個工具都會說：**這個工具在幹嘛、解決誰的什麼痛點、直銷商 會看到什麼、Pilot 想先驗證的事**。
> Pilot 階段不寫死量化門檻（可能還會調整），先確認方向對、再 GA 訂死。

---

### 3.1 關係記憶活檔案（MVP 差異化核心）

| 項目                        | 描述                                                                                                                                                                                             |
| :-------------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **這個工具在幹嘛**    | 給每位客戶建一份「活檔案」：基本資料、健康關注、家庭、工作、興趣、溝通偏好、最近 5 次互動、生活事件、標籤。資料不用一次填完，直銷商 邊聊邊累積                                                   |
| **解決誰的什麼痛點**  | •**Amy 痛點 ①**：客戶超過 50 人後記不住 context`<br>`• **Nina 痛點 ③**：親友資料零散在 LINE 對話、便利貼，沒地方放                                                             |
| **直銷商 會看到什麼** | • 每位客戶有獨立詳情頁，欄位可空、可慢慢補`<br>`• 4 種補資料入口：貼上對話、上傳截圖、語音備忘、Share Sheet`<br>`• 高價值欄位（如健康關注）缺漏時會跳「下次聊天順便問」提醒               |
| **不會做什麼**        | 不去爬 LINE / WhatsApp 對話歷史（隱私底線），全程由 直銷商 主動把資料丟進來                                                                                                                      |
| **Pilot 想先驗證**    | • 大部分客戶都有被建檔（基本三項資料以上）`<br>`• 手填單欄位夠快，直銷商 不會嫌麻煩`<br>`• 4 種補資料方式 直銷商 真的會用，不是擺好看的`<br>`• 不同 直銷商 之間資料 0 串到 — 這是底線 |

### 3.2 生活事件雷達

| 項目                        | 描述                                                                                                                                                                    |
| :-------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **這個工具在幹嘛**    | 主動偵測客戶的生活事件 — 生日、紀念日、寶寶出生、工作變動、健康話題 — 然後跳一張**非銷售**的關懷提醒給 直銷商                                                   |
| **解決誰的什麼痛點**  | •**Amy 目標 ⑤**：每週要做 10 個 Care Action，而不是 10 個推銷`<br>`• **共通價值**：客戶感受到「她記得我」，而不是「她又要賣東西」                      |
| **直銷商 會看到什麼** | • 每張提醒會說「為什麼今天跳出來」（例：「下週日是 Bella 生日」）`<br>`• 在 App 首頁的「今日 5 件事」會優先排這類卡                                                 |
| **Pilot 想先驗證**    | • 生日、紀念日這種「日期事件」不會漏`<br>`• 從對話抽取的事件（如「老公剛換工作」）大致是對的，直銷商 不用一直改`<br>`• 直銷商 收到提醒後願意採取行動，而不是滑掉 |

### 3.3 語氣／情緒感測器（精簡版）

| 項目                        | 描述                                                                                                                                                                             |
| :-------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **這個工具在幹嘛**    | 讀對方近期傳來的訊息，判斷對方現在是「壓力大／中性／開心」三檔，然後告訴 直銷商：「今天適不適合提產品」                                                                          |
| **解決誰的什麼痛點**  | •**Amy 痛點 ③**：拿捏不到朋友 tone vs 業務員 tone `<br>`• **防護價值**：避免 直銷商 在客戶心情低落時硬推產品、毀掉關係                                          |
| **直銷商 會看到什麼** | • 客戶詳情頁頂部一個小徽章：綠（適合聊）/ 黃（中性）/ 紅（今天先別賣）`<br>`• 生草稿前自動跑一次，紅燈時草稿語氣自動切「關懷」`<br>`• 直銷商 不同意可以一鍵改回想要的語氣 |
| **精簡版意思**        | v1 只分三檔，不細分情緒光譜、不做歷史趨勢圖。先看 客戶 願不願意聽建議，再決定要不要做厚                                                                                          |
| **Pilot 想先驗證**    | • AI 判斷方向跟真人感覺一致（不要綠燈時人覺得對方在哭、紅燈時人覺得對方在嗨）`<br>`• 紅燈跳出來時，直銷商 真的會切成關懷而不是硬推產品                                       |

### 3.4 太業務員警報（精簡版）

| 項目                        | 描述                                                                                                                                                                                                                       |
| :-------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **這個工具在幹嘛**    | 盯著 直銷商 對同一位客戶最近幾則訊息，連續 3 則都在賣東西（提到產品、價格、連結、樣品邀約）就跳警告：「先寄一張純關懷訊息再賣」                                                                                            |
| **解決誰的什麼痛點**  | •**反向守住 Amy 目標 ⑤**（每週要 10 個 Care Action 不是 10 個推銷）`<br>`• **守住 Linda 痛點 ④**：下線踩線她要承擔，AI 先把第一道關`<br>`• **品牌信任**：避免 直銷商 不自覺把每段對話都變成業配 |

## 4. 使用情境（直銷商 一天會發生什麼）

> 用真實的客戶名稱跑一遍。情境之間有連動 — 一個動作會觸發下一個工具，整個 App 是有機體不是功能堆疊。

### 情境 1 — 用截圖把 LINE 對話塞進關懷檔案

Amy 剛跟客戶 Anna 在 LINE 聊完，Anna 提到「最近睡不好」。Amy 不想手打抄寫，在 Anna 的詳情頁點「+ 從截圖補資料」→ 選擇截圖。
幾秒後系統認出來「睡眠改善」這個健康關注，Amy 確認套用。
**連動發生**：生活事件雷達跳一張提醒「下次提醒可關懷 Anna 睡眠主題」。

### 情境 2 — 生日前 7 天自動進今日 5 件事

Bella 的生日是 5/24，今天是 5/17。早上 Amy 開 App，今日 5 件事第一張就是：「下週日（5/24）是 Bella 生日 — 寫關懷訊息」。Amy 點進去直接生草稿。

### 情境 3 — 對方剛說「最近很累」，AI 建議今天先別推產品

Anna 剛回「最近工作好忙，覺得快爆炸了」。Amy 點「✨ 生成草稿」，情緒感測器先跳紅燈：「她今天壓力大，先別提產品 — 純關懷就好」。訊息草稿直接預設「關懷」語氣，連商業 tab 都收起來。

### 情境 4 — 連續 3 則在賣東西，AI 強制提醒切回關懷

Amy 過去 7 天傳給 David 的訊息：(1) 新品推薦 (2) 樣品邀約 (3) 折扣提醒。她想生第 4 則訊息時，太業務員警報彈窗：「你已連 3 則在賣東西，先寄純關懷訊息再賣」並預生成一段純關懷草稿：「David 上次說週末要陪兒子去爬山，有去成嗎？」

### 情境 5 — 早晨 10 秒內知道今天該做什麼

Amy 有 87 個客戶。06:45 開 App，今日 5 件事跳：(1) Bella 生日 (2) Cathy 樣品 48 小時 (3) David 補關懷 (4) Eric 30 天沉睡 (5) Fiona 生日 + 樣品合併。

### 情境 6 — 一鍵生 3 種語氣草稿全部過合規

Anna 的活檔案有「睡眠改善」「老同學關係」。Amy 點「✨ 生成訊息草稿 → LINE → 關懷」。秒回 3 種語氣草稿，全部過合規綠燈。選一版，點「複製到 LINE」開啟分享。

### 情境 7 — 高風險詞被合規低語擋下

Amy 想寫「保證一週瘦 5 公斤」。打字當下合規低語紅燈：「這句話有療效保證疑慮，建議改寫成『我自己用了一陣子，覺得作息和食慾有變化，分享給你參考』」。送出按鈕被禁用，必須採納改寫才能繼續。

### 情境 8 — 樣品發出 48 小時自動跳提醒

Amy 5/15 早上 10:00 發樣品給 Cathy 並 log 進系統。5/17 早上 10:00 今日 5 件事自動跳：「Cathy 樣品 2 天了，要不要關心一下？」並預生跟進草稿。

### 情境 9 — 把文字草稿一鍵變成語音

Amy 為 Anna 生了一段關懷草稿。點「🔊 生語音」→ 選溫暖女聲、繁中。10 秒內生成 25 秒 mp3，預覽確認後下載，切到 LINE 上傳給 Anna。Freemium 提示「今日剩 2/3 段語音額度」。

### 情境 10 — 客戶說「我先生反對」，異議處理器給 3 種接法

Amy 跟 Cathy 聊到加入團隊，Cathy 說「我先生不太能接受 MLM」。Amy 點「🛟 異議處理 → 家人不支持」。秒回 3 個選項：

- **共情型**：「我懂這個感覺，我先生一開始也是這樣⋯⋯」
- **提問型**：「請問他主要擔心的是時間還是金錢？」
- **邀請型**：「要不要找一天我們三個一起喝杯咖啡，讓他直接問？」
  Amy 選提問型 → 一鍵帶進草稿區微調。

### 情境 11 — 客戶主動想諮詢，直銷商 發問卷

Anna 主動說「我想知道我適合什麼產品」。Amy 點「📋 發問卷」→ 系統生 7 天有效連結。Anna 填完 12 題 → 結果回填到活檔案；摘要建議「主要關注睡眠與壓力管理，切角：睡眠主題關懷而非產品銷售」並寫進今日 5 件事新卡。

### 情境 12 — 招募話術被合規低語擋下

Amy 寫「加入我們團隊，月入 10 萬不是夢」。合規低語紅燈：「收入保證屬於 FTC 紅線，改寫成『我們團隊有些夥伴做得不錯，每個人成果不同⋯⋯』」。漏斗階段不前進直到改寫。

### 情境 13 — 客戶要求查詢／刪除自己的資料

Eric 透過 Amy 提出資料請求。Amy 在客戶詳情頁點「資料請求 → 匯出」。系統 30 天內匯出 Eric 100% 資料；如果是刪除請求，7 天內生效。

### 情境 14 — 不同 直銷商 的客戶資料絕對不串

R001、R002 同屬 Synergy 但不同租戶。R002 嘗試查 R001 的客戶 → 系統回 404 找不到。資料隔離是底線，紅隊測試必過。

```
┌─────────────────────────────────────────────────────────┐
│  前端：Mobile PWA（React 19 + Vite + Tailwind v4）       │

└────────────────────────┬────────────────────────────────┘
                         │ HTTPS REST + SSE streaming
┌────────────────────────▼────────────────────────────────┐
│  後端：FastAPI (uv) / Python 3.12                        │
│  ├─ 活檔案 Insight Agent（Sonnet 4.6）                  │
│  ├─ 生活事件雷達（規則 + Haiku 4.5）                     │
│  ├─ 情緒感測器（Haiku 三檔分類）                         │
│  ├─ 太業務員警報（純規則引擎）                           │
│  ├─ 今日 5 件事（規則 v1）                               │
│  ├─ 訊息草稿（Haiku 首字 + Sonnet 全文 streaming）       │
│  ├─ 樣品追蹤（Haiku 草稿 + 排程）                        │
│  ├─ 語音草稿（OpenAI TTS / ElevenLabs）                  │
│  ├─ 異議處理器（Haiku 速度優先）                         │
│  ├─ 健康問卷（Sonnet 摘要）                              │
│  ├─ 招募漏斗（Sonnet）                                   │
│  └─ 合規低語（regex sidecar < 50ms）                     │
└────────────────────────┬────────────────────────────────┘
                         │
        ┌────────────────┼────────────────┐
        ▼                ▼                ▼
   PostgreSQL +    Anthropic API    合規紀錄
   pgvector       (Haiku/Sonnet)    + 學習紀錄
   (Supabase RLS  + OpenAI TTS      + 觀測（OTel/Langfuse）
    租戶隔離)     成本上限           + 錯誤監控（Sentry）
                  ≤ $0.30/直銷商/日
```

**關鍵架構決策**：

- 資料庫用 PostgreSQL + pgvector，Supabase 提供租戶隔離
- AI 編排層用 LangGraph，並包一層內部抽象避免被 vendor 綁死
- 合規檢查 v1 純規則（< 50ms），之後再加 LLM 二審
- 多品牌支援：資料庫從 Day 1 就抽象「品牌」概念，但 v1 不建品牌後台 UI
- 學習紀錄從 Day 1 啟用 — 情緒判斷、警示採納、草稿採用都會留下來，給 Phase II 訓練模型用
- 不依賴 LINE 官方 API；直銷商 用四種補資料方式自己把資料丟進來
- 工程深度章節抽出為 Phase II/III 藍圖
